## Contributing
We welcome contributions! To propose changes:
1. Open an issue describing the problem or enhancement (if one does not already exist).
2. Create a feature branch:
   - git checkout -b feat/<short-description>
3. Make your changes with clear, incremental commits. Prefer Conventional Commits where practical (e.g., feat:, fix:, docs:, chore:).
4. Push your branch and open a Pull Request (PR) describing:
   - What changed and why
   - Any follow-up work needed
   - Screenshots or examples if applicable

### PR Checklist
- [ ] Linked to related issue(s) (e.g., Closes #<id>)
- [ ] Clear title and description
- [ ] Documentation updated as needed
